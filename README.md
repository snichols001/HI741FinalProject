# Clinical Data Warehouse

A role-based desktop application for managing and analyzing clinical patient data, built with Python and Tkinter. Users log in with their credentials and are presented with only the actions their role permits.

Final Project for HI 741 (Essential Programming for Health Informatics) at the University of Wisconsin–Milwaukee.

---

## Install

### Prerequisites

- Python 3.10 or later
- pip or conda

### Steps

**1. Clone the repository**

```bash
git clone https://github.com/snichols001/HI741FinalProject.git
cd HI741FinalProject
```

**2. Install dependencies**

Using pip:
```bash
pip install -r requirements.txt
```


Using conda:
```bash
conda install matplotlib=3.7.4
```

> **Note:** All other dependencies are included in the Python standard library (e.g., tkinter, csv, random, datetime, etc.).

**3. Generate the sample dataset (first run only)**

```bash
cd data
python ../data_generator.py
cd ..
```

This creates the seven CSV files the application reads on startup.

**4. Launch the application**

```bash
python src/main.py
```

## Usage

When the application starts, a login screen is shown. Enter a username and password from `data/credentials.csv`. The menu that appears depends on your role.

### Role permissions

| Role              | Available actions |
|-------------------|-------------------|
| clinician / nurse | Retrieve Patient, Add Patient, Remove Patient, Count Visits, View Note |
| admin             | Count Visits, Monitor Workload |
| management        | Generate Key Statistics, Monitor Revenue |

Clinician and nurse roles can access Protected Health Information (PHI). Admin and management roles cannot.

## Documentation

### Actions

- **Retrieve Patient**: enter a patient ID to display demographics and the most recent encounter
- **Add Patient**: enter a patient ID to check if the patient exists; if so, add a new encounter; if not, fill in demographics and create a new record. Changes are saved to `data/patients.csv` immediately.
- **Remove Patient**: enter a patient ID to delete all associated records after confirmation. Updates `data/patients.csv`.
- **Count Visits**: enter a date (YYYY-MM-DD) to see total encounters for that day, broken down by department. Clinicians and nurses also see a per-patient breakdown.
- **View Note**: enter a patient ID and date to read clinical notes from that visit
- **Generate Key Statistics**: displays four charts: age distribution, gender distribution, BMI distribution, and encounter type breakdown
- **Monitor Revenue**: shows total procedure costs per department
- **Monitor Workload**: ranks all providers by number of encounters handled

### Output files

| File | Description |
|------|-------------|
| `output/usage_log.csv` | One row per event: timestamp, username, role, action, and whether it succeeded. Failed login attempts are recorded here. |
| `data/patients.csv` | Updated in place whenever a patient is added or removed |

## Development

All source files live in `src/`. Run the program from the project root so relative paths resolve correctly:

```bash
python src/main.py
```

To add a new user role:

1. Add the role name and its allowed action keys to `ROLE_ACTIONS` in `src/user.py`
2. Update `can_access_phi()` in the same file if the role should access PHI
3. Add the role and credentials to `data/credentials.csv`

To check code quality:

```bash
cd src
python -m pylint *.py
```
